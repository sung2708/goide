# Frontend Design Skill

## Description
Create distinctive, production-grade frontend interfaces with high design quality.
Use this skill when the user asks to build web components, pages, or applications.
Generate creative, polished code that avoids generic AI aesthetics.

## Overview
This skill guides the creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics.
Always implement real, working code with exceptional attention to aesthetic details and creative choices.

## Input
The user provides frontend requirements, such as:
- Component

- Interface

- Purpose

---

## Design Thinking

Before coding, define a **clear and bold aesthetic direction**:

### 1. Purpose
- What problem does this interface solve?
- Who are the users?

### 2. Tone (choose a strong direction)
Examples:
- Brutally minimal
- Maximalist chaos
- Retro-futuristic
- Organic / natural
- Luxury / refined
- Playful / toy-like
- Editorial / magazine
- Brutalist / raw
- Art deco / geometric
- Soft / pastel
- Industrial / utilitarian

👉 Choose ONE strong direction and commit fully.

### 3. Constraints
- Framework (React, Vue, HTML, etc.)
- Performance requirements
- Accessibility

### 4. Differentiation
- What makes this design unforgettable?
- What is the ONE memorable element?

> CRITICAL: Intentionality > intensity

## Implementation Requirements

Your output must be:

- Production-grade and functional
- Visually striking and memorable
- Cohesive with a strong aesthetic identity
- Carefully refined in every detail

---

## Frontend Aesthetic Guidelines

### Typography
- Use distinctive, characterful fonts
- Avoid: Arial, Inter, Roboto, system fonts
- Combine:
  - One display font (unique)
  - One refined body font

---

### Color & Theme
- Use CSS variables for consistency
- Favor bold, dominant colors with sharp accents
- Avoid safe, evenly distributed palettes

---

### Motion
- Use animations intentionally
- Prefer CSS animations when possible
- Focus on:
  - Page load sequences (staggered reveals)
  - Hover interactions
  - Scroll-triggered effects

---

### Spatial Composition
- Avoid standard layouts
- Use:
  - Asymmetry
  - Overlapping elements
  - Diagonal flows
  - Grid-breaking layouts
- Balance:
  - Generous whitespace OR
  - Controlled density

---

### Backgrounds & Visual Details
Create depth and atmosphere using:
- Gradient meshes
- Noise textures
- Geometric patterns
- Transparency layers
- Dramatic shadows
- Decorative borders
- Custom cursors
- Grain overlays

---

## Anti-Patterns (STRICTLY AVOID)

❌ Generic AI aesthetics:
- Inter / Roboto / Arial
- Purple gradients on white backgrounds
- Predictable layouts
- Cookie-cutter components

❌ Repeating the same design style across outputs

---

## Creativity Rules

- Interpret requirements creatively
- Make unexpected design choices
- Vary:
  - Light and dark themes
  - Typography systems
  - Layout structures
- Never converge to common patterns

---

## Complexity Matching

- Maximalist designs → complex code, rich animations
- Minimalist designs → precision, spacing, typography

> Elegance comes from executing the vision correctly
